# 仕様書: Web UI

## 1. 概要

ユーザーがエージェントを操作するためのWeb UI。
FastAPIでHTMLを配信し、REST API + SSEで通信する。

**この仕様の範囲**: 3ペインレイアウト、全4タブの画面構成、ナビゲーション、モバイル対応
**範囲外**: 各タブのバックエンド処理（spec_chat.md, spec_runner.md, spec_config.md, spec_settings.md, spec_trigger.md を参照）

**対応デバイス**: PC（デスクトップブラウザ）、モバイル（スマートフォンブラウザ）
**使用技術**: Python (FastAPI + uvicorn), HTML, CSS, JavaScript (バニラ)

## 2. 補足資料

### 2.1 モックアップ

HTMLモックアップ: `docs/mockup_phase3.html`

### 2.2 参照ドキュメント
- `docs/spec_config.md` — エージェント定義、AgentInfo
- `docs/spec_chat.md` — チャット処理、SSEイベント形式
- `docs/spec_runner.md` — 自律思考、ログ・成果物API
- `docs/spec_settings.md` — 設定編集API
- `docs/spec_trigger.md` — トリガーAPI

### 2.3 データ構造とUIの対応

UIはエージェントのデータ構造に1対1で対応する。

```
エージェント名           ← 左ペイン
  ├─ 会話履歴 → 会話     ← 中ペイン（タブ+リスト） → 右ペイン（詳細）
  ├─ 作業履歴 → 作業
  ├─ タスク（ミッション概要 + タスク一覧）
  ├─ レポート一覧 → レポート
  └─ 設定 (CLAUDE.md, config.yaml)
```

## 3. 画面構成

### 3.1 3ペインレイアウト

```
┌──────────┬──────────────────┬──────────────────┐
│ 左ペイン   │ 中ペイン           │ 右ペイン           │
│ (20%)     │ (40%)             │ (40%)             │
│           │                   │                   │
│ エージェント│ タブ + リスト/内容  │ 選択項目の詳細     │
│ 一覧       │                   │                   │
└──────────┴──────────────────┴──────────────────┘
```

**ペイン比率**: 20 : 40 : 40（中↔右ペイン間にリサイズハンドルあり、ドラッグで比率変更可能）

**右ペインの保持**: 中ペインのタブを切り替えても、右ペインの表示は維持される。チャットを開いたまま、タスクやログを確認できる。

#### モバイル対応（768px以下）

モバイル画面では3ペインを同時に表示せず、1ペインずつ切替表示する。

- 全ペインが `100vw` × `100dvh` の全画面表示
- 左ペイン → エージェント選択 → 中ペインへスライド遷移
- 中ペイン → アイテム選択 → 右ペインへスライド遷移
- 各ペイン上部に「← 戻る」ボタンで前のペインに戻る
- リサイズハンドルは非表示
- `100dvh` を使用してモバイルブラウザのアドレスバー分のオーバースクロールを防止

### 3.2 左ペイン（エージェント選択）

- ロゴ「kobito_agent」
- セクションタイトル「エージェント」
- エージェント一覧（名前 + 説明）
- 選択中のエージェントはハイライト表示（左ボーダー + 背景色）
- ページ読み込み時に `GET /api/agents` で取得
- 最初のエージェントを自動選択

### 3.3 中ペイン（タブ + コンテンツ）

上部にタブバー。タブは以下の4つ:

| タブ | 表示形式 | 内容 |
|------|---------|------|
| 会話 | リスト型 | 会話一覧 + 新規会話ボタン |
| 作業 | リスト型 | 自律作業実行ボタン + 作業履歴一覧 |
| タスク | 複合型 | ミッション概要（折りたたみ）+ タスク一覧（インライン展開） |
| レポート | リスト型 | ファイル一覧（ファイル名 + サイズ） |
| 設定 | 直接表示型 | config.yaml編集 + CLAUDE.md編集 |

**リスト型タブ**: リストアイテムをクリックすると、右ペインに詳細が表示される。
**複合型タブ**: 中ペイン内で完結する。右ペインへの遷移はしない。
**直接表示型タブ**: 中ペインに内容を直接表示する。右ペインは変更しない。

### 3.4 右ペイン（詳細表示）

中ペインのリストで選択した項目の詳細を表示する。

| 元タブ | 右ペインの内容 |
|--------|-------------|
| 会話 | チャット画面（メッセージ履歴 + 入力欄） |
| 作業 | 作業ログ詳細（実行済/失敗バッジ、プロンプト、応答） |
| レポート | Markdownプレビュー |

**注意**: タスクタブは右ペインを使用しない。タスク詳細は中ペイン内でインライン展開される。

## 4. タブ別仕様

### 4.1 会話タブ

#### 中ペイン
- **新規会話ボタン**: 右ペインのチャット画面をクリアし、新しい会話を開始
- **会話一覧**: エージェント選択時に `GET /api/agents/{id}/conversations` で取得
  - 各行: 更新日時 + メッセージ件数 + タイトル（あれば）or 最後のメッセージ（切り詰め）
  - タイトルは要約実行後に表示される（会話データの `title` フィールド）
  - 選択中の会話はハイライト表示
  - 最新の会話を自動選択
- **新規会話の仮アイテム**: メッセージ送信時に会話一覧の先頭に仮アイテム（`conv-in-progress`）を追加。別画面に移動しても戻れるようにする
  - プレビューにユーザーのメッセージを表示
  - `onConversationId` でIDが来たら紐付け
  - 応答完了後に `loadConversations()` で正式な一覧に置き換わる

#### 右ペイン（チャット画面）
- **チャットヘッダー**（会話選択時に表示）
  - 1行目: タイトル（あれば）or 日時 + アクションボタン
  - 2行目: 要約（あれば、全文表示）
  - **アクションボタン**:
    - 「要約する」: `POST /api/agents/{id}/conversations/{conv_id}/summarize` を呼び、タイトル+要約を生成・保存。実行中は「要約中...」表示
    - 「CLI起動」: `POST /api/agents/{id}/launch-cli` でClaude CLIをターミナル起動（session_idがあれば --resume）
    - 「削除」: 確認ダイアログ後に `DELETE /api/agents/{id}/conversations/{conv_id}` で削除
- **メッセージ表示**: ユーザーは右寄せ、エージェントは左寄せ
- 各メッセージに送信者名ラベル（ユーザー: 「あなた」、エージェント: エージェント名）
- 各メッセージにタイムスタンプを表示
- エージェント応答はMarkdown（marked.js）でHTMLにレンダリング
- 新しいメッセージ表示時に自動スクロール
- ストリーミング中はカーソルアニメーション表示
- **入力エリア**: テキストエリア + 送信ボタン
  - Enter で送信、Shift+Enter で改行
  - 送信中は入力を無効化
  - 空メッセージは送信不可

#### 会話データ構造
```json
{
  "conversation_id": "uuid",
  "agent_id": "adam",
  "created_at": "ISO8601",
  "updated_at": "ISO8601",
  "session_id": "claude session id or null",
  "title": "要約で生成されたタイトル or null",
  "summary": "要約で生成された要約文 or null",
  "messages": [
    { "role": "user", "content": "...", "timestamp": "ISO8601", "source": "web" }
  ]
}
```

#### ストリーミング中の画面遷移
- 送信中のチャット状態（ユーザーメッセージ、エージェント名、bubble参照、蓄積テキスト）を `streamingChat` 変数に保持
- 別の会話を選択しても自由に切り替え可能
- `conv-in-progress` をクリックして戻ると、`streamingChat` のデータから再描画し、bubble参照を差し替え。以降の `onChunk` は新しいbubbleに追記される
- エージェント切り替え時に送信中なら確認ダイアログを表示

#### 処理フロー（メッセージ送信）
1. 入力テキストを取得し、入力欄をクリア
2. 新規会話なら会話一覧に仮アイテムを追加
3. `streamingChat` にストリーミング状態を保持
4. ユーザーメッセージを右ペインに即座に追加
5. `POST /api/agents/{id}/chat` をfetchで送信
6. `conversation_id` イベント受信 → 内部状態を更新
7. `chunk` イベント受信 → `streamingChat.bubble` にテキストを追加（30文字チャンク分割）
8. `done` イベント受信 → `streamingChat` クリア、入力を再有効化、中ペインの会話一覧を更新

#### 処理フロー（要約）
1. 「要約する」ボタンをクリック
2. `POST /api/agents/{id}/conversations/{conv_id}/summarize` を呼び出し
3. バックエンドが会話内容をテキスト化し、`Runner.summarize_text()` でLLMにタイトル+要約を生成させる
4. 生成結果を会話データの `title` / `summary` フィールドに保存
5. 右ペインのヘッダーにタイトル+要約を表示
6. 中ペインの会話一覧を更新（タイトルが反映される）

### 4.2 作業タブ

#### 中ペイン
- **「自律作業を１つ実行」ボタン**: `POST /api/agents/{id}/think` を呼び出し。セッション管理は自動（セッションファイルがあれば継続、なければ新規）
- **「プロンプト」ボタン**: 右ペインに作業プロンプト（`think_prompt.md`）の編集UIを表示
  - デフォルトプロンプトが表示され、カスタマイズ可能
  - `GET/PUT /api/agents/{id}/think-prompt` で取得・保存
- **作業実行中**:
  - ボタンを「作業中...」に変更し無効化
  - 作業履歴一覧の先頭に「作業中」（パルスアニメーション）のアイテムを追加
  - プレビューに最新のイベント（ツール名やテキスト）をリアルタイム更新
  - アイテムをクリックすると右ペインにストリーミング画面を表示（別画面から戻れる）
  - 完了後にボタンを元に戻し、作業履歴一覧を再読み込み
- **作業履歴一覧**: `GET /api/agents/{id}/logs` で取得
  - 各行: 実行済/失敗バッジ + 時刻 + 要約（切り詰め）
  - 選択中のログはハイライト表示
  - **同一セッションの統合**: 同じsession_idで複数回実行した場合、ログファイルは1つに統合される（eventsは追記、responseは最新で上書き）

#### セッション管理
- タスク未指定: `.think_session_id` ファイルでセッション管理。ファイルがあれば自動継続
- タスク指定: `.session_{task_filename}` ファイルでタスク単位にセッション管理。同じタスクなら同じセッションで継続
- `resume` パラメータは不要（バックエンドがセッションファイルの有無で自動判断）

#### 右ペイン（ストリーミング中）
- 「作業中」バッジ（パルスアニメーション）
- プロンプト（▶ クリックで展開/折りたたみ、デフォルト非表示）
- リアルタイムログ: ツール使用イベント（⚙アイコン + 緑色）とテキストイベントが順次追加
- 完了時: バッジが「実行済」/「失敗」に変わり、結果セクションに報告テキストを表示

#### 右ペイン（作業詳細 — ログ履歴）
- 実行済/失敗バッジ + タイムスタンプ
- プロンプト（▶ クリックで展開/折りたたみ）
- 実行過程: `events` 配列からツール使用・テキストを再現表示（ストリーミング中と同じ見た目）
- 結果: 報告テキスト
- 失敗時: エラーメッセージ

#### ストリーミング中の画面遷移
- ストリーミングイベントは `thinkStreamEvents` 配列に保持される
- 別の画面に行ってもイベントは裏で蓄積され続ける
- 作業履歴の「作業中」アイテムをクリックすると、蓄積されたイベントから画面を再構築
- 作業中にClaude Codeが `task.md` / `mission.md` を変更した場合、タスクタブのミッション表示を自動更新

### 4.3 タスクタブ

ミッション概要とタスク一覧を1つのタブに統合。右ペインへの遷移はしない。

#### 中ペイン

**ミッション概要（折りたたみ）**
- タブ最上部にミッションの1行概要を表示（mission.mdの最初のテキスト行）
- ▶ クリックで詳細を展開/折りたたみ（デフォルト: 折りたたみ）
- 展開時に以下を表示:
  - **「ミッションについて話す」ボタン**: 右ペインで新規会話を開始し、ミッション/タスクについてエージェントと対話する
    - mission.mdがある場合: 「mission.md と task.md を読んで、現状の課題や次にやるべきことについて話そう」を送信
    - mission.mdがない場合: 「まだ mission.md がないので、一緒にミッションを考えよう。CLAUDE.md を読んで、自分の役割からミッションを提案して」を送信
  - **「編集」ボタン**: 表示モードと編集モードを切り替え
  - **表示モード**: ミッション (mission.md) + タスク (task.md) をMarkdownレンダリング
  - **編集モード**: テキストエリアでMarkdownを直接編集。「保存」「キャンセル」ボタン
    - 保存: `PUT /api/agents/{id}/mission` + `PUT /api/agents/{id}/task` で書き込み
    - キャンセル: 変更を破棄して表示モードに戻る
- mission.md / task.md はエージェント選択時の `GET /api/agents/{id}` レスポンス内の `mission` / `task` フィールドから取得

**タスク一覧**
- ミッション概要の下にタスク一覧を表示: `GET /api/agents/{id}/tasks` で取得
- 各行: ステータスバッジ（未承認/承認済/完了）+ タイトル + プログレスバー + 進捗率
- **承認済タスクに「作業開始」ボタン**: クリックすると作業タブに切り替わり、そのタスクのfilenameを指定して `POST /api/agents/{id}/think?task={filename}` で自律作業を実行。タスク単位のセッションIDで継続管理される
- タスクをクリックすると、そのアイテムの下にインラインで詳細を展開（`GET /api/agents/{id}/tasks/{filename}` で取得、Markdownレンダリング）
- **未承認タスクの承認**: インライン詳細の上部に「承認する」ボタンを表示。クリックで `POST /api/agents/{id}/tasks/{filename}/approve` を呼び、ファイル内の `**ステータス: 未承認**` を `**ステータス: 承認済**` に書き換え。タスク一覧を再読み込み
- 再度クリックで折りたたみ。別タスクをクリックすると前の展開は閉じる
- 右ペインへの遷移はしない

#### 右ペイン
変更しない（前回の表示を保持）

### 4.4 レポートタブ

#### 中ペイン
- **ファイル一覧**: `GET /api/agents/{id}/outputs` で取得
  - 各行: ファイル名 + ファイルサイズ
  - 選択中のファイルはハイライト表示

#### 右ペイン（成果物プレビュー）
- `GET /api/agents/{id}/outputs/{filename}` で内容を取得
- MarkdownをHTMLにレンダリングして表示

### 4.5 設定タブ

#### 中ペイン（直接表示）
- **基本設定 (config.yaml)**
  - 名前（テキスト入力、必須）
  - モデル（テキスト入力、必須）
  - 説明（テキスト入力、任意）
- **トリガー設定**
  - 「定期トリガーを有効にする」チェックボックス
  - cron式（テキスト入力、5フィールド形式。チェックボックス有効時のみ表示）
  - ヒント: 「5フィールド形式（分 時 日 月 曜日）。例: */10 * * * *（10分ごと）」
  - トリガー状態表示: 実行状態（実行中/待機中）、次回実行時刻、最終実行時刻
  - API: `PUT /api/agents/{id}/trigger-config`, `DELETE /api/agents/{id}/trigger-config`
- **システムプロンプト (CLAUDE.md)**
  - テキストエリア（大、等幅フォント、任意）
  - ヒント: 「Markdown形式で記述できます」
- **ボタン**: 「保存」「リセット」

#### 右ペイン
変更しない（前回の表示を保持）

## 5. 処理フロー（共通）

### ページ読み込み時
1. `GET /api/agents` でエージェント一覧を取得
2. 左ペインにエージェント一覧を表示
3. 最初のエージェントを自動選択（→ エージェント選択時の処理へ）

### エージェント選択時
1. 左ペインの選択状態を更新（ハイライト）
2. チャットタブの会話一覧を `GET /api/agents/{id}/conversations` で取得
3. 会話がある場合、最新の会話を自動選択し右ペインに表示
4. 会話がない場合、右ペインをクリア
5. 中ペインの現在のタブに応じて内容を更新

### タブ切り替え時
1. 中ペインのコンテンツを切り替える
2. **右ペインは変更しない**（現在の表示を保持）

## 6. サーバー構成

```python
# server/app.py — エントリーポイント
app = FastAPI()
# APIルーター + 静的ファイル配信
app.mount("/", StaticFiles(directory="server/static", html=True))
```

起動: `start.bat` でuvicornを起動（ポート8300、ホットリロード有効）

### 静的ファイル構成

```
server/static/
  index.html        # SPA（シングルページ）
  css/
    style.css       # スタイル
  js/
    app.js          # メインアプリケーション
    api.js          # API通信ユーティリティ
    chat.js         # チャット表示機能
    settings.js     # 設定タブの表示・保存処理
    think-tab.js    # 思考タブの表示処理
```

## 7. 非機能要件

- ホットリロード: uvicornの `--reload` で、サーバーコード変更時に自動再起動
- Markdownレンダリングにはmarked.js（CDN）を使用
- SPA的に1ページで完結。ルーティングは不要
- モバイル対応: 768px以下で1ペイン切替表示（`100dvh` でオーバースクロール防止）
- 認証・認可は行わない（ローカル利用前提）
- ダークモードのみ

## 8. テスト方針

- Playwright MCPでE2Eテストを行う
- テスト用エージェントを作成し、サーバーを起動した状態でテストする
- LLM APIはテスト時もモックする
