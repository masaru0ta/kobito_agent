# 仕様書: Web UI

## 1. 概要

ユーザーがエージェントを操作するためのWeb UI。
FastAPIでHTMLを配信し、REST API + SSEで通信する。

**この仕様の範囲**: 3ペインレイアウト、全5タブの画面構成、ナビゲーション
**範囲外**: 各タブのバックエンド処理（spec_chat.md, spec_runner.md, spec_config.md, spec_settings.md, spec_trigger.md を参照）

**対応デバイス**: PC（デスクトップブラウザ）
**使用技術**: Python (FastAPI + uvicorn), HTML, CSS, JavaScript (バニラ)

## 2. 補足資料

### 2.1 モックアップ

### 2.2 参照ドキュメント

### 2.3 データ構造とUIの対応

UIはエージェントのデータ構造に1対1で対応する。

```
エージェント名           ← 左ペイン
  ├─ チャット履歴 → チャット  ← 中ペイン（タブ+リスト） → 右ペイン（詳細）
  ├─ 思考履歴 → 思考
  ├─ ミッション → タスク
  ├─ 成果物一覧 → 成果物
  └─ 設定 (CLAUDE.md, config.yaml)
```

## 3. 画面構成

### 3.1 3ペインレイアウト

```
┌──────────┬──────────────────┬──────────────────┐
│ 左ペイン   │ 中ペイン           │ 右ペイン           │
│ (20%)     │ (40%)             │ (40%)             │
│           │                   │                   │
│ エージェント│ タブ + リスト/内容  │ 選択項目の詳細     │
│ 一覧       │                   │                   │
└──────────┴──────────────────┴──────────────────┘
```

**ペイン比率**: 20 : 40 : 40

**右ペインの保持**: 中ペインのタブを切り替えても、右ペインの表示は維持される。チャットを開いたまま、ミッションやログを確認できる。

### 3.2 左ペイン（エージェント選択）

- ロゴ「kobito_agent」
- セクションタイトル「エージェント」
- エージェント一覧（名前 + 説明）
- 選択中のエージェントはハイライト表示（左ボーダー + 背景色）
- ページ読み込み時に `GET /api/agents` で取得
- 最初のエージェントを自動選択

### 3.3 中ペイン（タブ + コンテンツ）

上部にタブバー。タブは以下の5つ:

| タブ | 表示形式 | 内容 |
|------|---------|------|
| チャット | リスト型 | 会話一覧 + 新規会話ボタン |
| 思考 | リスト型 | 思考実行ボタン + 思考履歴一覧 |
| ミッション | 直接表示型 | mission.md + task.md |
| 成果物 | リスト型 | ファイル一覧（ファイル名 + サイズ） |
| 設定 | 直接表示型 | config.yaml編集 + CLAUDE.md編集 |

**リスト型タブ**: リストアイテムをクリックすると、右ペインに詳細が表示される。
**直接表示型タブ**: 中ペインに内容を直接表示する。右ペインは変更しない。

### 3.4 右ペイン（詳細表示）

中ペインのリストで選択した項目の詳細を表示する。

| 元タブ | 右ペインの内容 |
|--------|-------------|
| チャット | チャット画面（メッセージ履歴 + 入力欄） |
| 思考 | 思考ログ詳細（成功/失敗バッジ、プロンプト、応答） |
| 成果物 | Markdownプレビュー |

## 4. タブ別仕様

### 4.1 チャットタブ

#### 中ペイン
- **新規会話ボタン**: 右ペインのチャット画面をクリアし、新しい会話を開始
- **会話一覧**: エージェント選択時に `GET /api/agents/{id}/conversations` で取得
  - 各行: 更新日時 + 最後のメッセージ（切り詰め）
  - 選択中の会話はハイライト表示
  - 最新の会話を自動選択

#### 右ペイン（チャット画面）
- **メッセージ表示**: ユーザーは右寄せ、エージェントは左寄せ
- 各メッセージに送信者名ラベル（ユーザー: 「あなた」、エージェント: エージェント名）
- 各メッセージにタイムスタンプを表示
- エージェント応答はMarkdown（marked.js）でHTMLにレンダリング
- 新しいメッセージ表示時に自動スクロール
- ストリーミング中はカーソルアニメーション表示
- **入力エリア**: テキストエリア + 送信ボタン
  - Enter で送信、Shift+Enter で改行
  - 送信中は入力を無効化
  - 空メッセージは送信不可

#### 処理フロー（メッセージ送信）
1. 入力テキストを取得し、入力欄をクリア
2. ユーザーメッセージを右ペインに即座に追加
3. `POST /api/agents/{id}/chat` をfetchで送信
4. `conversation_id` イベント受信 → 内部状態を更新
5. `chunk` イベント受信 → エージェント応答にテキストを追加
6. `done` イベント受信 → 入力を再有効化、中ペインの会話一覧を更新

### 4.2 思考タブ

#### 中ペイン
- **思考実行ボタン**: クリックで `POST /api/agents/{id}/think` を呼び出し
  - 実行中はボタンを「思考中...」に変更し無効化
  - 完了後にボタンを元に戻し、思考履歴一覧を再読み込み
- **思考履歴一覧**: `GET /api/agents/{id}/logs` で取得
  - 各行: 成功/失敗バッジ + 時刻 + 要約（切り詰め）
  - 選択中のログはハイライト表示

#### 右ペイン（思考詳細）
- 成功/失敗バッジ + タイムスタンプ
- プロンプト（等幅フォント、折り返し）
- 応答（等幅フォント、折り返し）
- 失敗時: エラーメッセージを表示

### 4.3 ミッションタブ

#### 中ペイン（直接表示）
- **ミッション (mission.md)**: セクションラベル + 内容をMarkdownレンダリング
- **タスク (task.md)**: セクションラベル + 内容をMarkdownレンダリング（チェックボックス付き）
- mission.md / task.md はエージェント選択時の `GET /api/agents/{id}` レスポンス内の `mission` / `task` フィールドから取得

#### 右ペイン
変更しない（前回の表示を保持）

### 4.4 成果物タブ

#### 中ペイン
- **ファイル一覧**: `GET /api/agents/{id}/outputs` で取得
  - 各行: ファイル名 + ファイルサイズ
  - 選択中のファイルはハイライト表示

#### 右ペイン（成果物プレビュー）
- `GET /api/agents/{id}/outputs/{filename}` で内容を取得
- MarkdownをHTMLにレンダリングして表示

### 4.5 設定タブ

#### 中ペイン（直接表示）
- **基本設定 (config.yaml)**
  - 名前（テキスト入力、必須）
  - モデル（テキスト入力、必須）
  - 説明（テキスト入力、任意）
- **システムプロンプト (CLAUDE.md)**
  - テキストエリア（大、等幅フォント、任意）
  - ヒント: 「Markdown形式で記述できます」
- **ボタン**: 「保存」「リセット」

#### 右ペイン
変更しない（前回の表示を保持）

#### 処理フロー
- 詳細は `docs/spec_settings.md` を参照

## 5. 処理フロー（共通）

### ページ読み込み時
1. `GET /api/agents` でエージェント一覧を取得
2. 左ペインにエージェント一覧を表示
3. 最初のエージェントを自動選択（→ エージェント選択時の処理へ）

### エージェント選択時
1. 左ペインの選択状態を更新（ハイライト）
2. チャットタブの会話一覧を `GET /api/agents/{id}/conversations` で取得
3. 会話がある場合、最新の会話を自動選択し右ペインに表示
4. 会話がない場合、右ペインをクリア
5. 中ペインの現在のタブに応じて内容を更新

### タブ切り替え時
1. 中ペインのコンテンツを切り替える
2. **右ペインは変更しない**（現在の表示を保持）

## 6. サーバー構成

```python
# server/app.py — エントリーポイント
app = FastAPI()
# APIルーター + 静的ファイル配信
app.mount("/", StaticFiles(directory="server/static", html=True))
```

起動: `start.bat` でuvicornを起動（ポート8300、ホットリロード有効）

### 静的ファイル構成

```
server/static/
  index.html        # SPA（シングルページ）
  css/
    style.css       # スタイル
  js/
    app.js          # メインアプリケーション
    api.js          # API通信ユーティリティ
    chat.js         # チャット表示機能
    settings.js     # 設定タブの表示・保存処理
    think.js        # 思考タブの表示処理
```

## 7. 非機能要件

- ホットリロード: uvicornの `--reload` で、サーバーコード変更時に自動再起動
- Markdownレンダリングにはmarked.js（CDN）を使用
- SPA的に1ページで完結。ルーティングは不要
- モバイル対応は行わない
- 認証・認可は行わない（ローカル利用前提）
- ダークモードのみ
