# 仕様書: タスク連動自律思考

## 1. 概要

タスクファイル（`tasks/*.md`）と自律思考セッションをリンクさせる機能。
同じタスクに対する自律思考は1つのセッションにまとまり、思考の流れが保持される。
また、未承認タスクの実行をシステム側でブロックする。

**この仕様の範囲**: タスク一覧API、タスク指定思考実行、承認チェック、タスク単位セッション管理
**範囲外**: Web UIのタスク実行ボタン（別途実装）

**使用技術**: Python, FastAPI, pytest

## 3. 機能詳細

### 3.1 タスクファイル構造

各エージェントの `tasks/` ディレクトリに `.md` ファイルとして配置する。

```markdown
# タスクのタイトル

**ステータス: 承認済**

## チェックリスト
- [x] 完了した項目
- [ ] 未完了の項目
```

- タイトル: 先頭の `# ` 行から取得
- ステータス: 本文中の `**ステータス: xxx**` から取得。値は「未承認」「承認済」
- 進捗率: チェックリスト（`- [x]` / `- [ ]`）から自動計算
- `index.md` はタスク一覧から除外する

### 3.2 タスク一覧API

#### GET /api/agents/{agent_id}/tasks

タスク一覧を返す。

**レスポンス**:
```json
[
  {
    "filename": "task_001_example.md",
    "title": "タスクのタイトル",
    "status": "承認済",
    "progress": 50,
    "completed_tasks": 1,
    "total_tasks": 2,
    "date": "2026-03-27"
  }
]
```

**ソート順**: 未承認 → 承認済 → 完了の順。同ステータス内は進捗率の低い順。

#### GET /api/agents/{agent_id}/tasks/{filename}

タスクの詳細を返す。

**レスポンス**:
```json
{
  "filename": "task_001_example.md",
  "content": "Markdownの全文",
  "progress": 50,
  "completed_tasks": 1,
  "total_tasks": 2
}
```

### 3.3 タスク指定の自律思考

#### POST /api/agents/{agent_id}/think?task={filename}

指定タスクに対して自律思考を実行する。

**処理フロー**:

```
1. tasks/{filename} を読み込む
2. ステータスを確認
   - 「承認済」でなければ 403 Forbidden を返す
   - ファイルが存在しなければ 404 Not Found を返す
3. セッションIDを確認
   - .session_{filename} が存在すれば、そのIDで resume
   - 存在しなければ新規セッション
4. プロンプトを組み立てる
   - 新規: タスク内容を含む専用プロンプト
   - 続行: 短縮版プロンプト
5. claude -p を実行
6. 完了後、セッションIDを .session_{filename} に保存
```

#### タスク指定時のプロンプト（新規）

```
あなたは今から指定されたタスクを1ステップだけ進める。

## 対象タスク
{タスクファイルの内容}

## 手順
1. タスクのチェックリストを確認する
2. 未完了の項目から1つ選んで実行する
3. 完了したらチェックリストを更新する（tasks/{filename} を編集）
4. 必要に応じて成果物を output/ に保存する

## ルール
- 大きな項目は小さなステップに分解し、1ステップだけ進めること
- チェックリストの更新を忘れないこと
```

#### タスク指定時のプロンプト（続行）

```
前回の続きを1ステップだけ進めろ。
完了したらチェックリストを更新すること（tasks/{filename} を編集）。
```

#### セッションファイル

- パス: `agents/{name}/.session_{filename}`
- 例: `agents/adam/.session_task_002_e2e_testing.md`
- 内容: Claude Code のセッションID（テキスト）

### 3.4 承認チェック

- `**ステータス: xxx**` パターンを正規表現で抽出する
- `承認済` 以外（`未承認`、パターンなし等）は実行を拒否する
- エラーレスポンス: `403 Forbidden` `{"detail": "未承認のタスクは実行できません"}`

### 3.5 タスク未指定時の動作

`task` パラメータなしの場合は従来通りの動作を維持する。
`.think_session_id` による単一セッション管理、DEFAULT_THINK_PROMPT の使用。

## 4. 非機能要件

- 承認チェックはファイル読み込みのみで完結する（外部依存なし）
- セッションファイルはテキストファイル1行のみ

## 5. 考慮事項・制限事項

- ステータスの変更はタスクファイルの直接編集で行う（APIからの変更は別途実装）
- 完了判定の自動化（チェックリスト100%で自動的にステータス変更）は範囲外

## 6. テスト方針

- ユニットテスト（pytest）でAPI・ロジックをテストする
- LLM呼び出しはモックする
- タスクファイルはtmp_path上に作成する
- 承認チェックは正常系・異常系の両方をテストする

## 7. テスト項目

### タスク一覧API
- [ ] タスク一覧が返る（filename, title, status, progress等）
- [ ] index.mdが一覧に含まれない
- [ ] チェックリストから進捗率が正しく計算される（0%, 50%, 100%）
- [ ] ステータスが本文から正しく抽出される
- [ ] タスクディレクトリがない場合は空配列を返す
- [ ] 存在しないエージェントに対して404を返す

### タスク詳細API
- [ ] タスクの内容と進捗率が返る
- [ ] 存在しないファイルに404を返す

### 承認チェック
- [ ] 承認済タスクの実行が成功する
- [ ] 未承認タスクの実行が403で拒否される
- [ ] ステータス行がないタスクの実行が403で拒否される
- [ ] 存在しないタスクの実行が404を返す

### タスク単位セッション管理
- [ ] タスク指定思考の完了後、.session_{filename}にセッションIDが保存される
- [ ] 同じタスクの再実行で前回のセッションIDが使われる（resume）
- [ ] タスク指定なしの従来動作が維持される（.think_session_id使用）
