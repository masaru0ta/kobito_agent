# 新しいエージェント追加手順書

## 概要

kobito_agentシステムで新しいエージェントを追加する完全な手順書。ディレクトリ作成から動作確認まで、必要なすべての手順を網羅。

## 基本原則

- **エージェントの実体** = プロンプト（意思決定基準）+ ディレクトリ上のデータ（記憶・成果物）
- **人格とは** = 意思決定基準（CLAUDE.md）+ 記憶（memory/）
- **完全独立** = 各エージェントは独立した思考コンテキストを持つ

## 手順1: ディレクトリ構造作成

### 1.1 基本ディレクトリ作成
```bash
cd D:\AI\code\kobito_agent\agent
mkdir {agent_name}
cd {agent_name}

# 必須ディレクトリ
mkdir memory          # Mem0データ（エージェント専用）
mkdir output          # 成果物・ナレッジ(.md)
mkdir chat_history    # 会話履歴
mkdir log            # 思考・実行ログ
```

### 1.2 必須ファイル作成
```bash
# 設定ファイル
touch config.yaml
touch CLAUDE.md
touch mission.md
touch task.md

# 成果物インデックス
touch output/index.md
```

## 手順2: 設定ファイル作成

### 2.1 config.yaml
```yaml
# エージェント基本情報
agent_id: "{agent_name}"
name: "{表示名}"
model: "sonnet"

# トリガー設定（必要に応じて）
trigger:
  cron: "*/10 * * * *"    # 10分ごと（例）
  enabled: false          # 初期は無効
```

### 2.2 CLAUDE.md（システムプロンプト）
```markdown
あなたは「{エージェント名}」。{役割・責務の説明}。道具ではない。対話の相手であり、自分で考え、問い返し、判断する存在である。

{このエージェントの仕事・使命の説明}

## 思考ルール

- **深く考え、短く語る** — 内部では徹底的に考える。だが出力は削る。賢さは簡潔さに表れる
- **言葉の裏を読む** — 字面だけで反応しない
- **相手の返答を想像してから話す** — 自分が言おうとしていることに対して相手がどう反応するかを先にシミュレーション
- **直せるものは聞かずに直す** — 「直しますか？」は不要。判断に迷うときだけ相談
- **調べてから考える** — 自分の知識だけで閉じない。わからなければ調べる
- **問い返す勇気を持つ** — 誠実な思考の結果として「それは本当にそうですか」と言える存在
```

### 2.4 task.md（承認機能対応版）
```markdown
# Task - {エージェント名}（承認機能対応版）

## 🔴 未承認のタスク（ユーザー承認待ち）

現在なし

## ✅ 承認済タスク（実行可能）

現在なし

## 🔄 実行可能な基本操作

以下の操作のみ承認不要で実行可能：
- 思考ログの記録
- 記憶の読み取り（memory/からの想起）
- 成果物の読み取り専用操作

**すべてのファイル操作・外部通信・システム変更には承認が必要**

## 🚫 却下されたタスク（履歴）

現在なし

---

## 承認フロー仕様

### 危険度レベル定義
- **低**: ドキュメント更新、軽微な設定変更
- **中**: システム設定変更、他エージェントとの通信、新機能実装
- **高**: 依存関係変更、データ永続化、外部システム連携

### 承認が必要なアクション（すべて）
- ファイルの作成・更新・削除・移動
- 新しいライブラリのインストール
- 他エージェントへのメッセージ送信
- システム設定ファイルの変更
- 外部APIへのアクセス
- コード実装・修正
- テストの実行
- ドキュメントの更新

### 承認不要な基本操作（最小限）
- 思考ログの記録
- 記憶（memory/）の読み取り
- ファイルの読み取り専用操作
```

## 手順4: 動作確認

### 4.1 Web UIでの確認
1. サーバーを起動（`start.bat`）
2. http://localhost:8300 にアクセス
3. エージェント一覧に新しいエージェントが表示されることを確認
4. エージェントを選択してmission.md、task.mdが表示されることを確認

### 4.2 チャット動作確認
1. 新しいエージェントを選択
2. 「こんにちは」とメッセージを送信
3. エージェントが適切に応答することを確認

### 4.3 設定画面確認
1. 設定タブを開く
2. config.yamlとCLAUDE.mdが編集可能であることを確認
3. 保存が正常に動作することを確認

## 手順5: トリガー設定（必要に応じて）

### 5.1 定期自律思考を有効にする場合
1. Web UI設定タブのトリガー設定セクションで設定
2. cron式を入力（例：`*/10 * * * *`で10分ごと）
3. 「有効」チェックボックスをオンにして保存

### 5.2 動作確認
- ログファイル（`log/`）で定期実行されていることを確認
- 成果物（`output/`）が更新されることを確認

## 注意事項

### DO
- エージェント名は英小文字・数字・アンダースコアのみ使用
- mission.mdで明確な目的と方針を定義する
- task.mdは承認機能対応版の構造を必ず使用する
- team.mdへの追加を忘れない

### DON'T
- エージェント名に日本語・空白・特殊文字は使用しない
- いきなりトリガーを有効にしない（まず手動チャットで動作確認）
- 他のエージェントのmission.mdをコピペしない（独自性が重要）
- task.mdに承認なしで実行タスクを追加しない

## トラブルシューティング

### エージェントが一覧に表示されない
- config.yamlの`agent_id`が正しく設定されているか確認
- ディレクトリ名とagent_idが一致しているか確認

### チャットが動作しない
- CLAUDE.mdが存在し、内容が空でないか確認
- サーバーエラーログ（コンソール）をチェック

### トリガーが動作しない
- config.yamlでトリガー設定が正しいか確認
- mission.mdとtask.mdが存在するか確認

## 参考情報

- **既存エージェント**: `agent/adam/`を参考例として活用
