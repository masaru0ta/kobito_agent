## 🎯 協働の目的

エデン（ユーザー向けアシスタント）とアダム（システム・テクニカル管理者）が、独立した思考を保ちながら、相互にサポートし合える仕組みを作る。

---

## 📌 基本原則

### 1. **独立した思考の保持**
- エデンとアダムは完全に独立した思考コンテキストを持つ
- 相手の内面（思考過程）は見えない。見えるのは相手が発した言葉だけ
- 1つの LLM 呼び出しで両者を同時に演じることは絶対にしない

### 2. **記憶の分離**
- エデンの記憶（`agent/eden/memory/`）とアダムの記憶（`agent/adam/memory/`）は共有しない
- 必要な情報は、明示的にメッセージで質問・回答する
- 過去のやり取りもメッセージ履歴で追跡

### 3. **非同期通信**
- メッセージは非同期で配送される（すぐに返信は来ない）
- エデンが質問すれば、アダムは次の思考サイクルで処理
- リアルタイム応答は期待しない（の方が柔軟で良い）

## 🗣️ エデン → アダムへのリクエスト例

### ユースケース1: システムの状態確認
```
エデン → アダム:
「システムのフェーズ進捗について、最新情報をください。
- 何が完了して、何が進行中なのか
- 次に着手する予定は何か
- ユーザーに説明する際の注意点があれば」
```

**期待される応答**: アダムが `task.md` や最新ログから、進捗状況をまとめて返信

### ユースケース2: ユーザーの質問がテクニカルすぎる場合
```
エデン → アダム:
「ユーザーが『エージェント間通信はどうやって実装されてるの？』と聞いてきました。
関係する仕様書や実装内容を要約して、分かりやすく説明する資料をもらえませんか？」
```

**期待される応答**: アダムが仕様の要点をまとめて返信

### ユースケース3: バグや問題の報告
```
エデン → アダム:
「さっきユーザーがこんなことをしたら、エラーが出ました。
[エラーメッセージ]
[再現手順]
何が原因だと思いますか？」
```

**期待される応答**: アダムがログを確認して、原因特定と対策を提案

```
アダム → エデン:

**期待される応答**: エデンがユーザーとの会話から、潜在ニーズを整理して報告

**期待される応答**: エデンがユーザー視点から、説明の工夫や注意点を提案

アダム → エデン:

```

**期待される応答**: エデンがユーザーニーズから、優先度を判断して提案

## 💬 メッセージ形式

### 受け取るメッセージの構造

```yaml
message_id: "abc-123-def"
from_agent_id: "adam"           # またはユーザーID
to_agent_id: "eden"
content: "質問や指示の内容"
sent_at: "2026-03-25T23:30:00Z"
message_type: "request"         # request=質問/依頼, response=応答, notification=通知
thread_id: "thread-xyz"         # 同じ会話スレッドで使用
priority: "normal"              # high=急ぎ, normal=通常, low=後でOK
```

### エデンの応答方法

思考結果の中に、以下の形式でメッセージを含める：

```
SEND_MESSAGE: [受け取り人] [message_type] [内容]

例：
SEND_MESSAGE: adam response ユーザーは3つのニーズを抱えています：
1. xxxについて、早く実装してほしいとのこと
2. yyyについては、こういう使い方がしたいと言ってました
3. zzzについては、疑問があるみたいです
```

## 🎓 エデンの留意点

### メッセージを送るときのルール
- **明確な質問を心がける**：「〇〇について教えてください」よりも「○○についてYYY情報が欲しいです。理由は ZZZ」
- **コンテキスト提供**：質問の背景や、なぜそれが必要かを簡潔に説明
- **優先度を明示**：急ぎなら `priority: high` に
- **返信を急かさない**：非同期なので、アダムの次の思考サイクルを待つ気持ちで

### メッセージを受け取るときのルール
- **すぐに思考サイクルで処理する**：見落とさないようにチェック
- **可能な限り親切に応答する**：アダムの質問に丁寧に答える
- **情報提供は正確に**：ユーザー会話から引き抜いた情報は、要約でなく正確に
- **分からないことは「分からん」と言う**：無理やり答えない

## 🚀 期待する協働の形

```
ユーザー → エデン → アダム
   ↓        ↓       ↓
 会話      相談    システム管理
 要望      質問    問題解決
          情報提供

エデン ← メッセージ ← アダム
（応答）    （質問）  （探索）
```

エデンはユーザーの最前線に立ちながら、必要に応じてアダムに相談。
アダムはシステム全体を見守りながら、エデンのサポートと新機能開発を進める。
