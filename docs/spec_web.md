# 仕様書: 1-4 web

## 1. 概要

ユーザーがエージェントとチャットするためのWeb UI。
FastAPIでHTMLを配信し、REST API + SSEで通信する。

**この仕様の範囲**: サーバー起動、静的ファイル配信、チャット画面、エージェント選択
**範囲外**: 設定編集、成果物閲覧（Phase 2）、ログ閲覧（Phase 2）、コード修正UI

**対応デバイス**: PC（デスクトップブラウザ）
**使用技術**: Python (FastAPI + uvicorn), HTML, CSS, JavaScript (バニラ)

## 3. 機能詳細

### 3.1 サーバー構成

```python
# server/app.py — エントリーポイント
app = FastAPI()

# API ルーター
app.include_router(agents_router, prefix="/api")

# 静的ファイル配信
app.mount("/", StaticFiles(directory="server/static", html=True))
```

起動コマンド:
```bash
uvicorn server.app:app --reload --host 0.0.0.0 --port 8000
```

### 3.2 静的ファイル構成

```
server/static/
  index.html        # SPA（シングルページ）
  css/
    style.css       # スタイル
  js/
    app.js          # メインアプリケーション
    api.js          # API通信ユーティリティ
    chat.js         # チャット機能
```

### 3.3 画面要素

#### 左サイドバー（幅: 280px）

**ロゴ**
- 「kobito_agent」を表示

**エージェント一覧セクション**
- セクションタイトル「エージェント」
- エージェントごとに1行: 名前 + 説明（descriptionの先頭を表示）
- 選択中のエージェントはハイライト表示（左ボーダー + 背景色）
- ページ読み込み時に `GET /api/agents` で取得

**会話履歴セクション**
- セクションタイトル「会話履歴」
- 選択中エージェントの会話一覧を表示
- 各行: 更新日時 + 最後のメッセージ（切り詰め）
- 選択中の会話はハイライト表示（左ボーダー + 背景色）
- エージェント選択時に `GET /api/agents/{id}/conversations` で取得
- 会話クリックで右側に履歴を表示

#### 右メインエリア

**ヘッダー**
- エージェント名を表示
- エージェントのモデル名を表示（名前の下に小さく）
- 「新規会話」ボタン（右端）

**チャット表示エリア**
- メッセージを時系列で表示
- 各メッセージに送信者名ラベルを表示（ユーザー: 「あなた」、エージェント: エージェント名）
- 各メッセージにタイムスタンプを表示
- ユーザーメッセージ: 右寄せ、背景色付き
- エージェント応答: 左寄せ
- エージェント応答はMarkdownをHTMLにレンダリングする（軽量ライブラリ: marked.js）
- 新しいメッセージが来たら自動スクロール
- ストリーミング中はカーソルアニメーション表示

**メッセージ入力エリア**
- テキストエリア（複数行入力可能）
- 送信ボタン
- Enter で送信、Shift+Enter で改行（ヒントテキストを入力欄の下に表示）
- 送信中は入力を無効化する
- 空メッセージは送信不可

### 3.4 処理フロー

#### ページ読み込み時
1. `GET /api/agents` でエージェント一覧を取得
2. サイドバーにエージェント一覧を表示
3. 最初のエージェントを自動選択

#### エージェント選択時
1. 選択状態を更新（ハイライト）
2. `GET /api/agents/{id}/conversations` で会話一覧を取得
3. サイドバーの会話履歴セクションを更新
4. 会話履歴がある場合、最新の会話を自動で開く（会話選択時の処理を実行）
5. 会話履歴がない場合、チャット表示エリアをクリア（新規会話状態）

#### 会話選択時
1. `GET /api/agents/{id}/conversations/{conv_id}` で履歴を取得
2. チャット表示エリアに全メッセージを表示

#### メッセージ送信時
1. 入力テキストを取得し、入力欄をクリア
2. ユーザーメッセージをチャット表示エリアに即座に追加
3. `POST /api/agents/{id}/chat` をfetch（SSE）で送信
4. `conversation_id` イベント受信 → 内部状態を更新
5. `chunk` イベント受信 → エージェント応答欄にテキストを追加
6. `done` イベント受信 → 入力を再有効化、サイドバーの会話履歴を更新

#### 新規会話ボタン押下時
1. 現在の会話IDをクリア
2. チャット表示エリアをクリア
3. 入力欄にフォーカス

## 4. 非機能要件

- ホットリロード: uvicornの `--reload` で、サーバーコード変更時に自動再起動する
- 静的ファイルはブラウザキャッシュに依存する（Cache-Controlヘッダーは設定しない）
- Markdownレンダリングにはmarked.js（CDN）を使用する

## 5. 考慮事項・制限事項

- Phase 1ではSPA的に1ページで完結する。ルーティングは不要
- モバイル対応は行わない
- 認証・認可は行わない（ローカル利用前提）
- ダークモードのみ。ライトモード切り替えは対応しない
- 設定の編集機能はPhase 2で提供する

## 6. テスト方針

- Playwright MCPでE2Eテストを行う
- テスト用エージェントを作成し、サーバーを起動した状態でテストする
- LLM APIはテスト時もモックする（テスト環境ではモックサーバーまたはテスト用エンドポイントを使用）
- 画面要素の存在確認、クリック操作、テキスト入力、SSEストリーミング表示を検証する

## 7. テスト項目

### ページ読み込み
- ページにアクセスするとロゴ「kobito_agent」がサイドバー上部に表示される
- エージェント一覧がサイドバーに表示される（名前 + 説明）
- 最初のエージェントが自動選択される
- エージェント名がヘッダーに表示される
- エージェントのモデル名がヘッダーに表示される

### エージェント選択
- エージェントをクリックするとハイライト表示になる
- エージェントを切り替えるとヘッダーの名前が変わる
- エージェントを切り替えると会話履歴一覧が更新される
- エージェントを切り替えるとモデル名が更新される
- 会話履歴がある場合、最新の会話が自動で開かれる
- 会話履歴がない場合、チャット表示エリアがクリアされる

### メッセージ送信
- テキストを入力して送信ボタンをクリックするとメッセージが表示される
- ユーザーメッセージに送信者名「あなた」とタイムスタンプが表示される
- Enterキーでメッセージが送信される
- Shift+Enterで改行が入力される
- 送信中は入力欄と送信ボタンが無効化される
- エージェントの応答がストリーミングで表示される
- エージェント応答に送信者名（エージェント名）とタイムスタンプが表示される
- 応答完了後に入力欄が再有効化される
- 空のメッセージは送信できない（送信ボタンが無効）

### 会話履歴
- メッセージ送信後、サイドバーの会話履歴に新しい会話が表示される
- 会話履歴をクリックすると過去のメッセージが表示される
- 新規会話ボタンをクリックするとチャットがクリアされる

### Markdownレンダリング
- エージェント応答のMarkdown（見出し、リスト、コードブロック等）がHTMLに変換されて表示される

### レイアウト
- サイドバーとメインエリアが横並びで表示される
- ユーザーメッセージが右寄せで表示される
- エージェント応答が左寄せで表示される
- 新しいメッセージ表示時に自動スクロールする
